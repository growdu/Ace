## 1. 项目概述

### 1.1 项目背景

腾冲百分（又称千分、腰子分）是流行于云南保山、腾冲等地区的经典扑克牌游戏。本项目旨在设计一个跨平台的腾冲百分游戏平台，支持移动端、桌面端、Web端，支持用户注册、组队和机器人对战。

### 1.2 项目目标

- 开发一款跨平台（桌面端、Web端、移动端）的腾冲百分游戏
- 实现完整的用户系统和积分体系
- 支持多人在线组队对战
- 内置智能机器人，支持多种对战模式

### 1.3 技术选型

| 组件 | 技术方案 |
|------|----------|
| 客户端框架 | Tauri + React + TypeScript |
| 核心逻辑 | Rust |
| 后端 | Rust (Axum) + WebSocket |
| 数据库 | PostgreSQL |
| 实时通信 | WebSocket |
| 部署平台 | 云服务器 |

## 3. 游戏规则

### 3.1 基本信息

- **玩家人数**：4人，坐成方形，对家为搭档
- **使用牌数**：一副扑克牌（54张），包括大小王
- **基本目标**：庄家通过打牌获得自己叫的分数，闲家阻止庄家获得叫分

### 3.2 发牌与叫分

1. 顺时针发牌，每人发12张牌，剩余6张为底牌
2. 发牌后进行叫分，叫分范围：75分起叫，最高150分（根据约定）
3. 逆时针依次叫分，叫分最高者获得庄家身份
4. 庄家拿6张底牌，换牌后扣下6张底牌

### 3.3 牌型与比较

- **牌型**：仅允许出单张牌
- **牌点大小**：2 > A > K > Q > J > 10 > 9 > 8 > 7 > 6 > 5 > 4 > 3
- **花色大小**：黑桃 > 红桃 > 梅花 > 方块
- **比较规则**：
  - 先比较花色，花色大的牌更大
  - 花色相同时，再比较点数
  - 大王小王始终为主牌
  - 主牌花色可以大过其他花色
  - 首家出牌的花色是"本轮主色"，非主牌的其他花色不能大过它

### 3.4 分数牌

| 牌型 | 分值 |
|------|------|
| A | 10分 |
| 10 | 10分 |
| 5 | 5分 |

每副牌总分55分。

### 3.5 结算

- 每局独立结算，不累计
- 庄家获胜：获得分数 ≥ 所叫分数
- 庄家失败：获得分数 < 所叫分数，或被抠底

## 4. 功能设计

### 4.1 用户系统

#### 4.1.1 注册登录

| 功能 | 描述 |
|------|------|
| 用户名注册 | 支持用户名 + 密码注册 |
| 游客登录 | 临时游客身份，可 later 绑定账号 |
| Token 认证 | JWT Token 认证 |

#### 4.1.2 用户信息

| 字段 | 描述 |
|------|------|
| 用户名 | 唯一标识 |
| 头像 | 玩家自定义头像 |
| 等级 | 根据胜场数计算 |
| 积分 | 排位积分 |
| 胜场/负场 | 统计数据 |

### 4.2 组队与匹配

#### 4.2.1 匹配模式

| 模式 | 描述 |
|------|------|
| 单人匹配 | 随机匹配3名玩家 |
| 双人匹配 | 匹配队友 + 2名对手 |
| 人机匹配 | 匹配机器人作为队友 |
| 纯人机 | 1名玩家 + 3名机器人 |

#### 4.2.2 房间机制

- 4人房间，逆时针座位
- 房主可创建房间并邀请好友
- 等待玩家加入，超时自动开始

### 4.3 机器人系统

#### 4.3.1 机器人等级

| 等级 | 描述 |
|------|------|
| 简单 | 随机出牌，失误较多 |
| 普通 | 基础策略，偶尔失误 |
| 困难 | 高手策略，几乎无失误 |

#### 4.3.2 机器人场景

- 纯人机对战（玩家 vs 3个机器人）
- 混合对战（玩家 + 机器人 vs 玩家 + 机器人）
- 机器人替补（玩家掉线时由机器人接管）

## 6. 数据结构

### 6.1 数据库表

```sql
-- 用户表
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    avatar VARCHAR(255),
    level INTEGER DEFAULT 1,
    score INTEGER DEFAULT 1000,
    wins INTEGER DEFAULT 0,
    losses INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 游戏记录表
CREATE TABLE game_records (
    id SERIAL PRIMARY KEY,
    room_id VARCHAR(50) NOT NULL,
    player_ids INTEGER[],
    scores INTEGER[],
    winner_team INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 机器人配置表
CREATE TABLE bot_configs (
    id SERIAL PRIMARY KEY,
    level VARCHAR(20) NOT NULL,
    difficulty INTEGER NOT NULL
);
```
